🔩 Princípios de Implementação

🛡️
Validação de Entrada
Zod (TS) / Pydantic (Python)
Todo payload externo e interno passa por schema validation antes de qualquer lógica de negócio. Rejeição consistente com erro tipado e mensagem legível.
const schema = z.object({ ... });
const result = schema.safeParse(payload);
if (!result.success) throw new ValidationError(result.error);
🔁
Idempotência Universal
request_id + event_id obrigatórios
Middleware centralizado verifica chave de idempotência (Redis) antes de processar. Resposta cacheada para duplicatas. Garante exatamente uma execução.
const key = `idem:${request_id}`;
const cached = await redis.get(key);
if (cached) return JSON.parse(cached);
📊
Observabilidade Obrigatória
logs + métricas + traces
Cada implementação adiciona logs estruturados (JSON), incrementa counters Prometheus e propaga trace_id via OpenTelemetry. Sem observabilidade = bloqueado.
logger.info('action.executed', {
trace_id, tenant_id, duration_ms,
action: 'DEAL_CLOSED_WON' });
⚡
Resiliência por Padrão
timeout · retry · circuit breaker
Toda chamada externa usa timeout explícito, retry com backoff exponencial e circuit breaker (Resilience4j / cockatiel). Falhas transitórias recuperáveis > 95%.
const result = await withCircuitBreaker(
() => externalService.call(payload),
{ timeout: 5000, retries: 3 });
🔐
Multi-tenant Isolation
tenant_id em toda query
Toda query ao banco inclui WHERE tenant_id = :tenantId obrigatoriamente via middleware Prisma. Zero possibilidade de vazamento cross-tenant por esquecimento.
prisma.$use(async (params, next) => {
params.args.where = { ...params.args.where,
tenant_id: ctx.tenantId }; return next(params); });
🧪

happy path + edge cases + erros
Cada PR inclui testes unitários para lógica crítica, testes de integração para APIs e testes de contrato para interfaces entre serviços. Sem testes = sem merge.
